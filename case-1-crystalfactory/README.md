# Case 1 — CrystalFactory: оцифровка спецификаций из 1С

> Этот кейс — часть [тестового задания](../README.md). Общий отчёт пиши в `../REPORT.md`, не в отдельный файл этого кейса.

## Контекст

У меня кондитерское производство. Сейчас переношу данные из 1С УПП в новую ERP-систему. Я начал разбирать спецификации и номенклатуру, сделал пару штук как образец — но не успел закончить.

## Задача

1. **Разбери оставшиеся 5 изделий** из `data/bom_raw.csv` в формат JSON по образцу `data/example_done/bom_clean.json`. Сохрани результат в новый файл (например, `bom_clean_my.json`).

2. **Размeть оставшиеся 45 позиций** номенклатуры из `data/nomenclature_from_crystal.csv` в формат `data/example_done/nomenclature_clean.json` (добавь к 5 имеющимся).

3. **Залей данные в SQLite** базу `db/factory.db`. Схема уже есть (`schema.sql`), 2 изделия-образца уже залиты (`seed_examples.sql`). Дополни своими 5 изделиями и расширенной номенклатурой.

4. **Напиши 2-3 SQL-запроса** к итоговой базе. Какие именно — реши сам, исходя из того, что было бы полезно проверить (целостность данных, состав изделий, статистика). Положи их в файл `queries.sql`.

## Если что-то непонятно

**Разберись сам.** Если в данных явно есть ошибка — не игнорируй, запиши в общий REPORT.md. Если нужно принять решение (как объединить дубли, что делать с пустой единицей измерения) — прими и объясни почему.

## Что в кейсе

```
data/
  bom_raw.csv                       ← 5 изделий из 1С (твоя задача)
  nomenclature_from_crystal.csv     ← 50 строк номенклатуры
  example_done/
    bom_clean.json                  ← 2 разобранных изделия (образец формата)
    nomenclature_clean.json         ← 5 размеченных позиций (образец)
    bom_clean_source.csv            ← исходник этих 2 изделий
db/
  schema.sql                        ← схема SQLite
  seed_examples.sql                 ← наполнение базы из 2 образцов
  factory.db                        ← база с уже залитыми образцами
```

## Где могут быть «ловушки»

Это не подвохи — это реальные проблемы данных из 1С. Просто чтобы знал что бывает:

- Опечатки в названиях, которые приводят к **дублям** (одна и та же позиция под разными именами)
- **Несовпадение единиц измерения** между разными источниками
- **Многоуровневые BOM** (изделие → полуфабрикат → ингредиент)
- Лишние пробелы и **разнобой регистра** в названиях
- **Пустые поля** (нет цены, нет группы, нет единицы)

Что сделаешь с такими случаями — твоё решение, опиши в общем REPORT.md.
