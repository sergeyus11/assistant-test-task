# Тестовое задание — AI-помощник

Привет. Если ты дошёл до этого репо — спасибо за интерес к вакансии. Дальше — тестовое.

## Кто я и что ищу

Меня зовут Сергей. У меня кондитерское производство в Пензе и параллельно несколько небольших IT-проектов: стартап нутрициологии, сервис в блокчейн-сети, контент-блог, аналитический хаб. Всё это разное по сути, но управляю один. Мне нужен помощник, который **через AI-инструменты** (Claude Code, ChatGPT, Cursor) поможет со всеми этими разными задачами — не пишет код с нуля, а использует AI чтобы разбираться в данных, чистить таблицы, заводить контент, проверять расчёты, готовить документы.

Главное, что я хочу проверить этим тестом — **умеешь ли ты сам разобраться** в незнакомом домене. Я НЕ смотрю знаешь ли ты кондитерку, медицину или блокчейн. Я смотрю **подход**.

## Что внутри

Три коротких кейса из трёх разных моих проектов. Каждый в отдельной папке:

```
case-1-crystalfactory/    Кондитерка: разобрать данные из 1С, привести к формату
case-2-nutriai/           Контент: разобрать результаты обследования, написать карточку
case-3-gonka/             Tech docs: по коммитам в git собрать changelog
REPORT_TEMPLATE.md        Шаблон твоего общего отчёта по всем 3 кейсам
```

В каждой папке свой `README.md` с задачей, `data/` с входными файлами, `example_done/` с одним готовым примером (где это помогает).

## Сколько времени

Реальная работа — **3-5 часов**. Дедлайн 5 дней с момента получения. Если что-то занимает дольше 1.5-2 часов на один кейс — не страшно, просто опиши это в отчёте.

## Что прислать

1. **Свой git-патч** или ссылка на форк/гист
2. **`REPORT.md`** — общий отчёт по 3 кейсам: что сделал, что заметил, какие принял решения, какие вопросы остались (по каждому кейсу — отдельная секция)
3. **`prompts.md`** — главные промпты, которые ты давал AI (по 2-3 ключевых на кейс)
4. **Скринкаст 10-15 минут** — покажи **процесс**: как разбираешься, не только результат

## На что я смотрю

| | Что важно |
|---|---|
| Подхват контекста | Разобрался в незнакомом домене или сделал механически? |
| Странности в данных | В каждом кейсе намеренно есть «ловушки» — реальные проблемы данных, не подвохи. Заметил? Описал? |
| AI-промпты | Используешь как инструмент или как поисковик? Итеративный процесс или копипаста? |
| Аккуратность | Цифры сходятся? Единицы соответствуют? Сам себя проверил? |
| Принятие решений | Где данные противоречивы — принял решение и обосновал, или проигнорировал? |
| Отчёт | По REPORT.md видно как ты думаешь, или это формальная отписка? |
| Скринкаст | Видно как работаешь — мышление вслух, проверка результата AI, не паника при ошибках |

## Что не критично

- Скорость (3 часа или 6 — неважно)
- Красота кода / красота markdown
- Идеальный русский
- Знание Django / React / любого фреймворка
- Экспертиза в конкретном домене

## Если что-то непонятно

**Прими решение сам и обоснуй в REPORT.md.** В реальной работе тебе часто нужно будет работать с неполной информацией — это нормально. Если **точно** не можешь без уточнения — открой issue в этом репо с вопросом, я отвечу.
